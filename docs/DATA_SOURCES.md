## Future Import Requirements

- `dictionary-ne`: keep the exact package version, source tarball URL, license text, attribution notice, import script, ranked lexical expansion script, generated romanized alias script, and replacement instructions for the bundled `.aff`, `.dic`, and ranked wordlist data.
- `dictionary-ne` ranked expansion: run `npm run lexicon:rank-hunspell` and `npm run lexicon:merge` only against bundle-safe dictionary entries. Local Wikipedia frequency artifacts stay ignored and are documented as ranking research input, not bundled corpus data. Merge scripts must not blindly overwrite curated seed data.
- `dictionary-ne` alias review: run `npm run dictionary:review` to generate a review TSV from LGPL dictionary words. Do not promote aliases into runtime word packs until a human review marks rows as approved and source/license notices are kept.
- Frequency imports: run `npm run import:frequency -- --source <source-id> --input <path|-> [--limit <n>]`. Bundle-safe sources may write to `src/data/wordlists/`; blocked sources fail; local-research-only sources write only to ignored `data/generated/frequency/`.
- Real Preeti validation: follow `docs/REAL_PREETI_VALIDATION.md`; target 30-50 consented documents before any real-document quality claim.
- Varnavinyas: keep it behind a local development flag until bundle size, worker latency, false positives, and UI wording are measured against project fixtures.
