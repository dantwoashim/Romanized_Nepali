# Pilot Feedback Plan

Pilot feedback is required before public release. No real user documents are committed unless they are manually submitted, redacted, and consented.

## Pilot Groups

- Niraj/LTK reviewers
- traditional typists
- students
- teachers
- office users
- developers
- document-heavy users

## Feedback Form Fields

- mode
- input
- expected output
- current output
- app/platform
- whether protected tokens were involved
- screenshot optional
- consent checkbox
- redaction confirmation

## Metrics

- candidate acceptance rate
- correction acceptance
- ignored suggestion rate
- error reports
- latency reports
- mode preference
- Traditional layout complaints
- dictionary misses

## Rules

- Opt-in only.
- No hidden typed-text collection.
- Users submit examples manually.
- Examples must be redacted.
- Consent metadata is required.
