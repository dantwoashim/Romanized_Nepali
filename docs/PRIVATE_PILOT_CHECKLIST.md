# Private Pilot Checklist

Generated: 2026-05-28

Status: blocked until native dev builds are validated or the pilot is explicitly scoped to Keyboard Lab/companion demo.

## Before Pilot

- Verification loop passes.
- Scorecard is fresh.
- Windows/macOS native dev build is installed and tested, or pilot scope says “engine/lab only.”
- Privacy policy is shown.
- Consent form is ready.
- Redaction policy is ready.
- Crash/log export is redacted.

## Pilot Groups

- Niraj/LTK stakeholders.
- Traditional typists.
- office users.
- students.
- teachers.
- developers.
- document-heavy users.

## Pilot Tasks

- Type common Romanized words and office phrases.
- Try protected tokens such as emails, IDs, and file names.
- Try Traditional Unicode suggestions.
- Report Traditional physical layout mismatches only after audited build exists.
- Accept/reject suggestions and report wrong rankings.
- Use dictionary and memory controls.
- Export diagnostics after redaction.

## Exit Criteria

- No P0 privacy or corruption bug.
- No host-app freeze in native path.
- Candidate duplicates remain zero.
- Shortcuts remain sequential.
- At least one Traditional typist validates the physical layout before Traditional physical launch.
