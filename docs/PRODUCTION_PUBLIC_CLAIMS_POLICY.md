# Production Public Claims Policy

Generated: 2026-05-27

## Allowed Claims Now

These are allowed when phrased conservatively:

- local-first keyboard engine prototype;
- browser Keyboard Lab for validation;
- Romanized live typing prototype;
- Romanized helper suggestions prototype;
- proofread, dictionary, and memory prototype;
- Traditional layout source-of-truth audit is pending;
- Windows TSF and macOS IMK scaffolds exist;
- native release requires platform work, signing/notarization, and pilot feedback.

## Forbidden Claims Until Evidence Exists

Do not claim:

- production Windows keyboard complete;
- production macOS keyboard complete;
- signed/notarized release available;
- complete LTK replacement shipped;
- government-ready;
- 100% accurate;
- best Nepali keyboard;
- beats Gboard;
- beats Hamro;
- production-ready native release;
- real-document superiority without consented test samples.

## Required Evidence For Stronger Claims

| Claim type | Required evidence |
| --- | --- |
| Native Windows ready | TSF implementation tested in target apps, installer tested, signing available, pass-through verified |
| Native macOS ready | IMK/XPC implementation tested, notarization available, install/uninstall tested, pass-through verified |
| Traditional layout ready | audited keymap, source provenance, typist validation, fixtures passing |
| Public beta ready | crash handling, privacy review, pilot feedback, support path |
| Accuracy claims | scoped fixtures, hostile suites, real consented samples, no hidden failures |

## External Blockers

- Windows code-signing certificate.
- Apple Developer ID and notarization.
- Real native Windows/macOS test environments.
- Human LTK/Traditional layout validation.
- Human pilot feedback.
- Safe licensed dictionary meaning source.
